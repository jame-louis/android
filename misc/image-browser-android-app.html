创建 XML 布局
编写界面布局文件

LinearLayout 核心属性

属性 说明 常用值
android:orientation 子视图排列方向 "vertical", "horizontal"
android:layout_width 布局宽度 "match_parent", "wrap_content"
android:layout_height 布局高度 "match_parent", "wrap_content"
android:padding 内边距 "16dp", "8dp"
android:gravity 子视图对齐方式 "center", "center_horizontal"

分步构建布局

2.1 根 LinearLayout

提示：创建一个垂直方向的 LinearLayout 填满整个屏幕，并添加一些内边距让界面更美观。

提示：TextView 常用属性：

提示：ImageView 关键属性：

添加图片资源
准备显示的图片文件

应用需要图片来显示。你需要将图片资源添加到项目中。

图片存放位置

将图片放在: res/drawable/

支持的格式：
  • PNG (推荐，支持透明)
  • JPG/JPEG (用于照片)
  • WebP (现代格式，体积更小)
  • Vector Drawables (XML 格式，可缩放)

命名规范

重要：Android 资源名必须：
  • 以小写字母开头
  • 只能包含小写字母 a-z、数字 0-9 和下划线
  • 不能有空格、大写字母

✅ 正确: image1.png, nature_photo.jpg, cat_01.png
❌ 错误: Image 1.png, 1st-image.jpg

测试用图

如果没有准备图片，可以使用 Android 内置图标：

<!-- 在 ImageView 中使用系统图标 -->
android:src="@android:drawable/ic_menu_gallery"
android:src="@android:drawable/ic_menu_camera"

处理按钮点击
实现上一张/下一张功能

现在使用 setOnClickListener 让按钮具有实际功能。

5.1 理解 OnClickListener

setOnClickListener 方法接收一个 OnClickListener 对象，定义按钮点击时的行为：

// setOnClickListener 语法模式
buttonReference.setOnClickListener(new View.OnClickListener() {
    @Override
    public void onClick(View v) {
        // 点击按钮时执行的代码
    }
});

5.2 实现"上一张"按钮逻辑

实现逻辑：
  1. 当前索引减 1
  2. 检查是否小于 0
  3. 如果是，循环到最后一张图片 (images.length - 1)
  4. 更新 ImageView 显示新图片
  5. 更新 TextView 显示新标题

5.3 实现"下一张"按钮逻辑

实现逻辑：
  1. 当前索引加 1
  2. 检查是否大于等于 images.length
  3. 如果是，循环到第一张图片 (0)
  4. 更新 ImageView 显示新图片
  5. 更新 TextView 显示新标题

5.4 创建辅助方法

由于两个按钮都需要更新显示，创建一个辅助方法：

// 辅助方法：更新显示
private void updateDisplay() {
    // 设置图片资源到 ImageView
    // 提示：使用 setImageResource()

    // 设置文本到 TextView
    // 提示：使用 setText()
}
别忘了！ 在 onCreate() 方法中调用 updateDisplay()，这样应用启动时会显示第一张图片。

Button - 常用属性

属性 说明 示例
android:text 按钮文字 "上一张", "下一张"
android:background 按钮背景 "@drawable/btn_bg"
android:id 唯一标识 "@+id/btnPrevious"

layout_weight 详解

layout_weight 属性对于响应式布局至关重要，它决定额外空间如何在子视图间分配。

工作原理：
  1. 水平方向设置 layout_width="0dp"，垂直方向设置 layout_height="0dp"
  2. 通过 weight 值按比例分配空间
  3. 相等 weight = 相等分配

示例：两个按钮都设置 layout_weight="1" 将平分空间 50-50。

测试与调试
排查常见问题

常见问题与解决方案

问题 可能原因 解决方案
应用启动崩溃 findViewById 在 setContentView 之前调用 确保 setContentView 先执行
图片不显示 资源名错误或不在 drawable 中 检查文件名匹配，代码中不加扩展名
按钮无响应 未设置 OnClickListener 确认调用了 setOnClickListener
按钮大小不一致 layout_weight 设置不正确 layout_width="0dp" 配合 layout_weight

调试技巧

  • 使用 Log.d("TAG", "message") 添加调试输出
  • 应用崩溃时查看 logcat 错误信息
  • 在 Android Studio 中使用断点逐步调试
  • 验证 R.id 引用与 XML 中的 ID 匹配
记住：修改 Java 代码后，可能需要重新构建项目 (Build → Make Project 或 Ctrl+F9)。

进阶练习提升技能

完成基础应用后，尝试以下扩展功能来练习你的技能：

初级挑战

1. 添加计数器显示

在按钮下方用 TextView 显示 "第 X 张，共 Y 张"。需要使用 + 运算符连接字符串。

2. 添加图片描述

创建另一个字符串数组存储图片描述，显示在标题下方。

3. 边界禁用按钮

不循环切换，而是在第一张禁用"上一张"按钮，最后一张禁用"下一张"按钮。使用 setEnabled(false)。

中级挑战

4. 添加滑动手势

研究 GestureDetector 并实现左右滑动切换图片。

5. 添加缩放功能

使用 PhotoView 库或为 ImageView 实现双指缩放。

6. 添加动画效果

切换图片时添加淡入淡出或滑动过渡。了解 ViewPropertyAnimator 或 ObjectAnimator。

高级挑战

7. 从网络加载图片

使用 Glide 或 Picasso 库从 URL 加载图片，而非本地资源。

8. 添加缩略图网格

添加第二个 Activity，用 GridView 显示所有缩略图，点击打开主视图对应位置。
